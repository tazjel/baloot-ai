# UI Standards & Design System

## Overview
Target aesthetic: "Premium, Dynamic, Modern". High-quality animations, glassmorphism, and responsive interactions.

*(To be populated based on current CSS)*

## Component Rules

### Buttons
- **Primary Actions**: distinct gradient or solid color.
- **Secondary Actions**: outlined or subtle background.

### Gameplay Elements
- **Sawa Button**:
  - **Visibility**: Only appears when cards in hand <= 4.
  - **Position**: Bottom center, distinguishable from play area.
- **Project Declarations**:
  - **Timing**: Triggered automatically or via specific phase prompts.
  - **Visuals**: Should have a "reveal" animation.

## Responsive Behavior
- **Hand Fan**: Cards should overlap more as screen width decreases, but never fully obscure the indices.
